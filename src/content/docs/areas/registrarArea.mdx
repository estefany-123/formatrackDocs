<h3>Encabezados de Solicitud</h3>

| Encabezado    | Valor            | Descripcion                                          |
| ------------- | ---------------- | ---------------------------------------------------- |
| Content-Type  | application/json | Indica que los datos esran manejados en fromato JSON |
| Authorization | Bearer `token`   | Token con autorizacion para acceder a las areas      |

<h3>Endpoint para los Datos del Área</h3>

Para agilizar la creacion de nuevas areas te ofrecemos estos endpoints de apoyo para que puedas consultar y obtener el ID unico tando de sedes como de usuarios, los cuales son necesarias para crear tu area.

<li>
  <a href="/sedes/listarSedes">Sede</a>
</li>
<li>
  <a href="/usuario/listarUsuarios">Usuario</a>
</li>

<h3>Parametros del cuerpo</h3>

| Parametro y Tipo    | Descripcion                                                                                                                                        |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| nombre (string)     | Nombre que se le va a designar a dicha area no mayor a (50)                                                                                        |
| estado (boolean)    | Si se encuentra activo o inactiva el area                                                                                                          |
| createdAt (date)    | fecha de creació del area.Esta se crea automaticamente                                                                                             |
| updatedAt (date)    | fecha de actualización del area.Esta se crea automaticamente                                                                                       |
| fk_sede (number)    | ID de la sede en la que esta el area. Para consultar las sedes existentes consulte este endpoint <a href="/sedes/listarSedes">sedes</a>            |
| fk_usuario (number) | ID del usuario al area perteneciente. Para consultar los usuario existentes consulte este endpoint <a href="/usuarios/listarUsuarios">usuarios</a> |

### Body

```json
{
  "nombre": "string",
  "estado": "boolean",
  "fk_sede": "number",
  "fk_usuario": "number"
}
```
