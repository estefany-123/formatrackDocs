Este endpoint permite registrar un nuevo rol con los datos proporcionados.

Method: `POST`

<Tabs>
  <TabItem label="Endpoint">
  ```
  http://localhost:3000/api/rol/register/
  ```
  </TabItem>
</Tabs>

<h3>Encabezados de Solicitud</h3>

| Encabezado | Valor | Descripcion
|----------|-------| -------
| Authorization |  Bearer `token` | Token con autorizacion para acceder a los roles

<h3>Parametros del cuerpo</h3>

| Parametro y Tipo | Descripcion
|----------|--------
| nombre (string) |  Nombre asignado del rol
| estado (boolean) | Si el rol esta activo o inactivo
| fecha_creacion (date) | Fecha en la que se registro el rol
| fecha_actualizacion (date) | Fecha en la que se actualizo el rol 

### Body

```json
{
  "nombre": "string",
  "estado": "boolean
  "fecha_creacion": "date",
  "fecha_actualizacion": "date",
}
```

<h3>Ejemplos de respuesta</h3>

<Tabs>
  <TabItem label="Status 201">
  ```json
  {message:"Rol registrado exitosamente"}
  ```
  </TabItem>

  <TabItem label="Status 400">
  ```json
  {message:"No fue posible registrar el rol"}
  ```
  </TabItem>
</Tabs>
