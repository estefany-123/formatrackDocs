<h2>¿Qué son los roles?</h2>

Los roles representan los distintos niveles de acceso y permisos que puede tener un usuario dentro del sistema. Cada rol determina qué módulos o funcionalidades están habilitadas para quienes lo poseen.

Por ejemplo, un usuario con el rol **Administrador** puede tener acceso completo al sistema, incluyendo la gestión de usuarios, configuraciones generales y reportes; mientras que un **Instructor** solo puede interactuar con funcionalidades relacionadas con su ficha de formación.

La asignación de roles es fundamental para garantizar la seguridad, la segmentación de tareas y el control de acceso dentro de la plataforma.

<h3>Descripción</h3>

Este endpoint permite obtener la lista de todos los roles registrados en el sistema.
